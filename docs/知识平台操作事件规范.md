# 知识平台操作事件规范

## 1. 目标

操作事件用于记录知识平台内每一次关键动作，形成可追溯、可汇总、可回放的操作链路。

## 2. 事件范围

建议纳入事件记录的动作包括：

- 原始文件上传
- 原始文件删除
- 原始文件回滚
- 预处理
- Chunk 生成
- Embedding 生成
- 向量索引构建
- 知识库初始化
- 知识库激活
- 知识库配置更新
- 知识库删除
- 检索测试
- 反馈记录

## 3. 事件字段

| 字段 | 说明 |
|---|---|
| `event_id` | 事件唯一 ID |
| `event_type` | 事件类型 |
| `knowledge_base_id` | 所属知识库 |
| `source` | 来源，例如页面、API、定时任务 |
| `actor` | 操作人或系统 |
| `input_assets` | 输入资产列表 |
| `output_assets` | 输出资产列表 |
| `params` | 事件参数快照 |
| `status` | success / failed / running |
| `started_at` | 开始时间 |
| `finished_at` | 完成时间 |
| `duration_ms` | 耗时 |
| `error_message` | 错误信息 |
| `log_path` | 详细日志路径 |
| `remark` | 备注 |
| `created_at` | 事件写入时间 |

## 4. 存储格式

推荐使用 `jsonl`：

- 一行一条事件
- 适合增量追加
- 适合按天聚合生成日报

建议目录：

```text
operations/
  events/
    2026-05-24.jsonl
```
