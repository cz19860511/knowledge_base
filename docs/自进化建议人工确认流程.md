# 自进化建议人工确认流程

## 1. 目标

把自进化建议从“系统输出”变成“人确认后再进入执行”的流程，保证平台不会越权自动改动关键配置或资料。

## 2. 决策类型

建议统一使用以下决策：

- `approve`：同意执行
- `reject`：拒绝执行
- `defer`：暂缓执行，后续再看

## 3. 人工确认字段

每条确认记录建议包含：

- 建议内容
- 建议来源日报/回放
- 决策类型
- 确认人
- 确认时间
- 备注

## 4. 当前实现

当前已提供：

- `GET /operations/evolution-confirmations`
- `POST /operations/evolution-confirmations`
- `GET /operations/evolution-confirmation-report`
- 确认记录会写入 `operations/evolution_confirmations/YYYY-MM-DD.jsonl`

## 5. 使用原则

- 所有建议默认都要人工确认。
- 确认结果必须留痕。
- `approve` 不等于自动执行，后续仍需进入明确的执行步骤。
- 对高风险建议，优先选择 `defer` 或 `reject`。
