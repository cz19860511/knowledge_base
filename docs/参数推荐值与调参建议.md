# 参数推荐值与调参建议

这是一份给现场直接用的速查版，只保留各参数的推荐默认值、常见调法和影响方向。

## 1. 默认推荐值

### MinerU 预处理

| 参数 | 推荐值 | 说明 |
|---|---|---|
| `folders` | `02-07` 全选 | 默认把当前标准化资料都纳入处理。 |
| `primary_exts` | `.pdf, .docx, .xlsx, .pptx` | 主解析文件类型。 |
| `supplement_exts` | `.docx` | 只给 DOCX 做补充解析。 |
| `markitdown_docx_enabled` | `true` | 保留 DOCX 备份文本稿。 |
| `mineru_command` | `mineru` | 默认命令名。 |
| `mineru_pipeline` | `pipeline` | 默认 MinerU pipeline 名称。 |

### Chunk 切分

| 参数 | 推荐值 | 说明 |
|---|---|---|
| `folders` | 与预处理一致 | 避免预处理和切分目录不一致。 |
| `max_chunk_chars` | `1800` | 中文场景比较稳的默认长度。 |
| `min_chunk_chars` | `300` | 防止 chunk 太碎。 |
| `mineru_parser_bonus` | `50` | 默认偏向 MinerU 结果。 |

### Embedding

| 参数 | 推荐值 | 说明 |
|---|---|---|
| `provider` | `transformers` | 正式推理模式。 |
| `model_path` | `/data/kb/models/bge-small-zh-v1.5` | 当前首版推荐模型。 |
| `model_name` | `bge-small-zh-v1.5` | 用于显示和日志标识。 |
| `device` | `cpu` | ECS 首版默认 CPU 跑。 |
| `batch_size` | `16` | 吞吐和稳定性比较平衡。 |
| `pooling` | `cls` | 首版先保持简单稳定。 |
| `query_instruction` | 默认中文检索前缀 | 让查询向量更贴近检索任务。 |
| `max_length` | `512` | 中文问答场景比较稳。 |
| `normalize` | `true` | 便于做余弦相似度检索。 |

## 2. 什么时候该调

### 先调目录，再调参数

最先改的是 `folders`。如果目录选错，后面的参数调得再细也没用。

### Chunk 相关

- 觉得检索结果太碎，先把 `max_chunk_chars` 调大。
- 觉得一段内容太长、命中不准，先把 `max_chunk_chars` 调小一点。
- 觉得很多极短 chunk 影响效果，先把 `min_chunk_chars` 调大一点。

### Embedding 相关

- 推理太慢，先把 `batch_size` 调小。
- 机器内存紧张，先把 `batch_size` 调小，再考虑 `max_length`。
- 如果后面换 GPU，再把 `device` 改成 `cuda`。
- 如果查询短句命中不理想，优先先改 `query_instruction`，不要先改模型。

## 3. 调参方向

### 想让召回更完整

- `max_chunk_chars` 调大一点
- `min_chunk_chars` 保持或略微调大
- `query_instruction` 保持启用

### 想让结果更精确

- `max_chunk_chars` 调小一点
- `min_chunk_chars` 不要太低
- `mineru_parser_bonus` 保持较高，让结构化结果优先

### 想让处理更稳

- 保持默认 `bge-small-zh-v1.5`
- 保持 `cpu + batch_size=16`
- 保持 `normalize=true`

## 4. 不建议轻易改的项

- `provider`：首版不要随便切到 `mock`
- `model_path`：除非已经确认新模型可用
- `pooling`：先别频繁切换
- `mineru_command`：只有命令路径不一致时才改

## 5. 一句话建议

首版就按这套来：

- `MinerU` 全目录跑
- `Chunk` 保持 `1800 / 300`
- `Embedding` 保持 `bge-small-zh-v1.5 + cpu + cls + normalize`

如果后面要调，优先顺序是：

1. `folders`
2. `max_chunk_chars`
3. `batch_size`
4. `query_instruction`
