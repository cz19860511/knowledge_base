# 大模型接入实施任务表

## 1. 目标

本任务表用于把《大模型接入知识平台方案》和《大模型接入接口清单与数据流图》落到可执行任务，按照三层结构推进：

- 业务问答层
- 平台运行层
- 自进化建议层

建议按“先稳定可用，再补运行记忆，最后做自进化建议”的顺序实施。

## 2. 任务拆解总览

| 阶段 | 任务 | 目标产物 | 依赖 | 优先级 |
|---|---|---|---|---|
| 0 | 统一接入边界 | 确定业务问答层、平台运行层、自进化建议层的输入输出 | 无 | 高 |
| 1 | 接通业务问答层 | AgentArts / 业务 Agent 能通过检索接口稳定拿到证据 | 现有 `retrieve` 接口 | 高 |
| 2 | 接通平台运行层 | 操作日志、日报、知识库状态可汇总可检索 | 事件记录、日报生成器 | 高 |
| 3 | 接通自进化建议层 | 能基于日志和日报输出优化建议 | 运行记忆库、业务知识库 | 中 |
| 4 | 建立回放与复盘机制 | 能从日志和日报复盘问题、定位根因 | 事件日志、资产版本 | 中 |
| 5 | 建立优化闭环 | 建议可转为任务，经过确认后执行 | 平台任务编排 | 中 |

## 4. 推荐实施顺序

### 第 1 轮

优先做以下任务：

- 1.1 固化检索接口契约
- 1.2 明确知识库路由方式
- 1.3 AgentArts 联调
- 2.1 定义操作事件 schema
- 2.2 接入原始文件事件

### 第 2 轮

接着做以下任务：

- 2.3 接入流程事件
- 2.4 接入知识库事件
- 2.5 生成日报模板
- 2.6 日报自动生成
- 2.8 日报自动调度
- 4.1 资产版本链设计

### 第 3 轮

最后做以下任务：

- 2.7 日报入库
- 3.1 到 3.5 自进化建议层
- 4.2 到 4.5 闭环与治理

## 5. 里程碑建议

| 里程碑 | 完成标准 |
|---|---|
| M1 | 大模型能稳定通过知识平台检索到业务证据 |
| M2 | 平台所有关键操作都有结构化事件日志 |
| M3 | 每日日报自动生成并进入平台运行记忆知识库 |
| M3a | 每日日报自动补跑，计划任务可持续工作 |
| M4 | 大模型能基于日报总结平台状态 |
| M5 | 大模型能给出参数调优、补资料、回滚等建议 |
| M6 | 建议可进入人工确认和执行闭环 |

## 6. 落地原则

- 先保证问答可用，再做运行记忆，再做自进化建议。
- 事件日志要结构化，日报要可读，建议要可执行但不可直接执行。
- 业务知识库与平台运行记忆库必须分离。
- 所有建议动作都要留人工确认入口。
