# 知识平台检索接口契约

## 1. 目标

本契约用于固化知识平台对外检索接口 `/knowledge-bases/retrieve` 的请求、返回、路由和兼容规则，作为 AgentArts、业务 Agent 和平台前端的统一调用规范。

## 2. 接口定义

- 路径：`POST /knowledge-bases/retrieve`
- 认证：`Authorization: Bearer <KB_API_KEY>`
- 作用：按照指定知识库或当前激活知识库检索证据片段，并返回可追溯结果

## 3. 路由规则

检索时的知识库选择优先级如下：

1. `knowledge_base_id`
2. `knowledge_base_ids`
3. 当前激活知识库 `active_knowledge_base_id`

更完整的路由定义见：[知识库路由规则说明.md](/Users/chenzhuo/hb/knowledge_base/docs/知识库路由规则说明.md)。

### 3.1 兼容规则

- `knowledge_base_id` 为首选字段，推荐新接入方优先使用。
- `knowledge_base_ids` 保留兼容，适合旧调用方式。
- 如果两者都传，`knowledge_base_id` 必须包含在 `knowledge_base_ids` 中，否则返回 `400`。
- 如果两者都不传，则默认使用当前激活知识库。

## 4. 请求字段

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---:|---|---|
| `knowledge_base_id` | `string` | 否 | `null` | 优先检索的知识库 ID |
| `knowledge_base_ids` | `string[]` | 否 | `[]` | 允许检索的知识库 ID 列表，兼容旧调用 |
| `query` | `string` | 是 | - | 查询内容，1~4000 字符 |
| `method` | `string` | 否 | `doc` | 当前固定为 `doc` |
| `offset` | `int` | 否 | `0` | 结果偏移，不能小于 0 |
| `limit` | `int` | 否 | `5` | 返回条数，建议 1~50 |
| `top_k` | `int` | 否 | `5` | 召回候选数，建议 1~50 |
| `search_threshold` | `float` | 否 | `null` | 命中阈值，范围 0~1 |
| `extra_params` | `[{key,value}]` | 否 | `[]` | 预留扩展字段 |

## 6. 行为约束

- 返回结果必须可追溯到 chunk 和原始来源。
- 当知识库没有索引或没有 chunk 时，应返回空结果而不是报错。
- 当知识库切换后，检索结果必须跟随当前知识库或显式指定的知识库。
- 检索接口不负责生成最终答案，只负责返回证据。

## 7. 推荐调用方式

### 7.1 新接入方

优先使用单库路由：

```json
{
  "knowledge_base_id": "ai_qna_standard_v1",
  "query": "服务区危化品车辆现场处理流程是什么",
  "top_k": 5,
  "limit": 5,
  "search_threshold": 0
}
```

### 7.2 旧接入方

如果暂时不能改造，只传 `knowledge_base_ids` 也可以：

```json
{
  "knowledge_base_ids": ["ai_qna_standard_v1"],
  "query": "服务区危化品车辆现场处理流程是什么"
}
```

## 8. 结论

本契约的核心是：

- 新接口优先用 `knowledge_base_id`
- 旧接口保留 `knowledge_base_ids`
- 检索必须有可追溯证据
- 空库、空索引时返回空结果
