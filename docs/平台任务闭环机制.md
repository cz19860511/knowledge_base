# 平台任务闭环机制

## 1. 目标

把自进化建议和人工确认结果转成可管理的任务台账，让建议进入“待办 - 执行 - 完成”闭环。

## 2. 任务来源

当前支持两种来源：

- 人工直接创建任务
- 从已确认的自进化建议生成任务

## 3. 任务状态

建议统一使用以下状态：

- `pending`：待处理
- `ready`：可执行
- `running`：执行中
- `blocked`：被阻塞
- `done`：已完成
- `cancelled`：已取消

状态流转建议遵循：

- `pending -> ready`
- `ready -> running`
- `running -> done`
- `running -> blocked`
- `blocked -> ready`
- `any -> cancelled`

## 4. 当前实现

当前已提供：

- `GET /operations/platform-tasks`
- `POST /operations/platform-tasks`
- `PUT /operations/platform-tasks/{task_id}`
- `GET /operations/platform-tasks/{task_id}`
- `POST /operations/platform-tasks/{task_id}/transition`
- `POST /operations/platform-tasks/from-confirmation`
- `GET /operations/platform-task-report`
- `GET /operations/platform-task-history`
- 任务台账写入 `operations/platform_tasks/YYYY-MM-DD.jsonl`
- 任务历史写入 `operations/platform_tasks_history/YYYY-MM-DD.jsonl`

## 5. 使用原则

- 每条建议先进入人工确认，再决定是否生成任务。
- 任务必须有状态和责任人。
- 任务完成后要回写状态。
- 任务台账只记录执行管理信息，不直接执行任何危险动作。
- 每次状态流转都要留历史。

## 6. 后续增强方向

- 增加任务详情页。
- 增加任务和建议的双向跳转。
- 增加任务执行日志。
- 增加任务到期提醒和周报汇总，并支持每日自动生成和手动补跑。
