# 知识平台日报规范

## 1. 目标

日报用于把当天的操作事件汇总成一份可读、可检索、可入库的 Markdown 文档，既给人看，也给大模型读。

## 2. 输出位置

建议输出到：

```text
operations/daily/YYYY-MM-DD.md
```

## 3. 推荐章节

- 今日概览
- 事件统计
- 关键事件
- 知识库分组
- 待关注事项

## 4. 生成原则

- 只写事实，不写长篇解释。
- 保留事件类型、状态、知识库 ID、参数摘要。
- 失败事件需要单独凸显。
- 日报保持可被后续预处理和向量化处理。

## 5. 当前实现

当前已具备：

- 按天聚合操作事件
- 输出 Markdown
- 支持按知识库过滤
- 支持手动脚本生成

## 6. 下一步

后续可以把日报自动接入平台运行记忆知识库，形成平台自进化的输入源。
