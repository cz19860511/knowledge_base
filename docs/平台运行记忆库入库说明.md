# 平台运行记忆库入库说明

## 1. 目标

将每日操作日报自动转成一个独立知识库，供大模型检索平台运行状态、失败记录和优化趋势。

## 2. 入库对象

入库对象是当天生成的 `operations/daily/YYYY-MM-DD.md`。

## 3. 目标知识库

- 知识库 ID：`platform_run_memory`
- 目标目录：`knowledge_bases/platform_run_memory`
- 目标 batch：`batch_20260521`

## 4. 生成结果

入库后会生成：

- `selected/batch_20260521/documents/YYYY-MM-DD/selected.md`
- `chunks/batch_20260521/chunks.jsonl`
- `vectors/batch_20260521/*`
- `rag/hybrid_build_summary.md`

## 5. 对外能力

入库完成后，可以直接通过知识平台检索：

- 平台当天发生了哪些操作
- 哪些步骤失败
- 哪些知识库最近变更较多
- 哪些参数或流程值得调整

## 6. 触发方式

### 6.1 API

`POST /operations/daily-report/ingest?date=YYYY-MM-DD`

### 6.2 脚本

```bash
python scripts/ingest_daily_report_to_memory.py --date YYYY-MM-DD
```

## 7. 后续建议

后续可以把这一步接入定时任务，让日报在每天生成后自动入库。
