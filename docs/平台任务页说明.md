# 平台任务页说明

平台任务页用于承接“建议确认 -> 任务台账 -> 状态流转 -> 历史回看”的执行闭环，入口为 `/platform-tasks-ui`。

## 页面内容

- 任务筛选：按日期、状态、来源筛选任务。
- 任务概览：展示当前筛选下的任务总数和状态分布。
- 导出 Markdown：把当前筛选结果导出为 Markdown。
- 保存任务报告：把当前筛选结果写入平台任务台账报告。
- 导出详情 Markdown：把当前选中的任务详情和历史导出为 Markdown。
- 保存历史报告：把当前日期的任务历史轨迹写入 Markdown 报告。
- 自动汇总状态：查看任务到期提醒和周报的自动调度状态。
- 立即补跑：手动触发任务到期提醒和周报的补跑。
- 从确认生成任务：把确认记录直接转成平台任务。
- 确认记录面板：直接浏览当日确认记录并一键转任务。
- 任务列表：展示标题、状态、负责人、截止日期和来源。
- 任务详情：展示任务主体、来源、备注和原始载荷。
- 状态流转：支持把任务推进到 `ready / running / blocked / done / cancelled`。
- 历史轨迹：查看任务创建、更新和状态流转记录。
- 执行日志：给当前任务补充处理进展、阻塞点和完成记录。
- 到期提醒与周报：查看逾期、即将到期、未设截止任务，并导出提醒和周报。

## 接口依赖

- `GET /operations/platform-tasks`
- `GET /operations/platform-tasks/{task_id}`
- `POST /operations/platform-tasks/{task_id}/transition`
- `GET /operations/platform-task-report`
- `GET /operations/platform-task-history`
- `GET /operations/platform-task-history-report`
- `GET /operations/platform-task-logs`
- `POST /operations/platform-tasks/{task_id}/logs`
- `GET /operations/platform-task-log-report`
- `GET /operations/platform-task-due-report`
- `GET /operations/platform-task-weekly-report`
- `GET /operations/platform-task-report/automation`
- `POST /operations/platform-task-report/automation/run`

## 使用方式

1. 打开页面后先输入 API Key。
2. 先用“从确认生成任务”把确认记录转成任务，或者直接加载任务列表。
3. 点击某条任务查看详情和历史。
4. 在右侧填写目标状态、负责人、截止日期和备注，点击“应用流转”。

## 说明

- 页面当前面向本地联调和平台内部使用，不做开放访问。
- 任务列表默认以当天日期为主，跨天查看时切换任务日期即可。
- 任务状态和历史会继续保存在平台任务台账中，便于后续对账和回放。
