# 平台总览页说明

## 1. 目标

把平台最常看的状态聚合到一页里，方便快速判断“平台现在是不是健康的、哪条链路有问题、最近有没有确认过建议”。

## 2. 页面内容

总览页会展示：

- 当前激活知识库
- 知识库总数
- 今日事件总数和失败数
- 今日日报、版本对账、事件回放的摘要
- 自进化建议总数和人工确认记录
- 平台任务总数和任务摘要
- 确认记录卡片可直接转成任务
- 最新的事件时间线

## 3. 页面入口

- 首页顶部入口
- `/platform-overview-ui`

## 4. 当前实现

当前总览页复用以下接口：

- `/health`
- `/knowledge-base-registry`
- `/knowledge-bases`
- `/operations/events`
- `/operations/daily-report`
- `/operations/daily-report/automation`
- `/operations/version-reconciliation`
- `/operations/replay-report`
- `/operations/evolution-suggestions`
- `/operations/evolution-confirmations`
- `/operations/platform-task-report`
- `/operations/platform-tasks`
- `/operations/platform-tasks/{task_id}`
- `/operations/platform-tasks/{task_id}/transition`
- `/operations/platform-task-history`
- `/operations/platform-tasks/from-confirmation`
- `/operations/assets`

## 5. 使用方式

- 先输入 API Key。
- 点击加载总览。
- 重点看失败事件、对账缺失、建议数量和确认记录。

## 6. 后续增强方向

- 增加趋势图和环比。
- 增加按知识库切换的总览过滤器。
- 增加错误高亮和告警状态。
- 增加可跳转到详细报告的按钮。
