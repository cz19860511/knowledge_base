# 平台任务自动汇总调度说明

平台任务自动汇总用于每天按固定时间自动生成两类产物：

- 到期提醒报告：汇总当前日期的逾期任务、即将到期任务和未设截止任务。
- 平台任务周报：汇总最近若干天的任务台账和状态分布。

## 默认行为

- 默认调度时间为 `00:20`。
- 默认检查间隔为 `60` 秒。
- 自动调度发现有待处理日期时，会按天补跑。
- 手动点击“立即补跑”时，会按当前页面日期强制生成一次到期提醒和周报。

## 相关配置

- `KB_PLATFORM_TASK_REPORT_RUN_TIME`
- `KB_PLATFORM_TASK_REPORT_CHECK_INTERVAL_SECONDS`

## 相关接口

- `GET /operations/platform-task-report/automation`
- `POST /operations/platform-task-report/automation/run`
- `GET /operations/platform-task-due-report`
- `GET /operations/platform-task-weekly-report`

## 产物目录

- `operations/platform_task_report_automation.json`
- `operations/platform_task_due/YYYY-MM-DD.md`
- `operations/platform_tasks_weekly/YYYY-MM-DD.md`

## 说明

- 自动汇总只负责生成提醒和周报，不会修改任务状态。
- 如果当天没有任务，也会写出空报告，方便留痕和排查。
