# 平台状态总结提示词

## 1. 目标

让大模型读取平台操作事件和日报后，生成简洁、可执行的状态总结。

## 2. 输入

建议输入以下内容：

- 当天操作事件统计
- 当天日报内容
- 自进化建议列表
- 最近 3 天失败事件趋势

## 3. 输出要求

输出建议包含：

- 今日总体状态
- 主要变更
- 失败和风险点
- 需要关注的目录或知识库
- 可执行的下一步建议

## 4. 提示词模板

```text
你是知识平台的运行复盘助手。

请根据下面的事件统计、日报和建议，输出一份简洁的状态总结。
要求：
1. 先总结平台状态，再列出风险。
2. 只基于给定材料，不要编造。
3. 每条建议都要说明依据。
4. 语言要简洁，面向平台负责人。

输入材料：
{{daily_report}}

{{event_summary}}

{{evolution_suggestions}}

输出格式：
1. 平台状态总览
2. 重点风险
3. 值得关注的知识库/目录
4. 建议动作
```

## 5. 使用方式

这份提示词后续可以作为“平台运行记忆知识库”的检索结果模板，让大模型做日常总结。
