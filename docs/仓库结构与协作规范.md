# 仓库结构与协作规范

## 目标

本仓库只管理知识平台的代码、配置和工程文档，不直接管理原始资料、解析产物、chunk、向量库、模型文件等大体量数据。

这样做有三个好处：

- 仓库轻量：方便 clone、review、部署和回滚。
- 数据安全：原始资料和模型不直接暴露在 Git 历史里。
- 边界清晰：代码版本、资料版本、向量库版本可以分别管理。

## 推荐目录

```text
knowledge_base/
├── docs/                    # 平台方案、技术选型、部署说明、协作规范
├── embedding_service/       # 独立 embedding 服务
├── kb_api/                  # AgentArts General 接口适配服务
├── scripts/                 # 资料预处理、chunk、建库、检索脚本
├── requirements-*.txt       # Python 依赖
├── raw/                     # 原始资料，不进 Git
├── staging/                 # 预处理输入暂存，不进 Git
├── working/                 # 解析、评估等过程产物，不进 Git
├── selected/                # 人工确认后的 selected.md，不进 Git
├── chunks/                  # chunk JSONL/统计结果，不进 Git
├── vectors/                 # keyword/embedding/hybrid 索引，不进 Git
├── rag/                     # RAG 测试结果，不进 Git
├── packages/                # 对外交付包，不进 Git
├── outputs/                 # Excel、预览、临时输出，不进 Git
└── models/                  # 本地或服务器模型文件，不进 Git
```

## Git 中应该保留

- `README.md`
- `.gitignore`
- `.dockerignore`
- `.gitattributes`
- `docs/**/*.md`
- `kb_api/**/*.py`
- `kb_api/Dockerfile`
- `kb_api/docker-compose*.yml`
- `kb_api/nginx.conf`
- `embedding_service/**/*.py`
- `embedding_service/Dockerfile`
- `scripts/**/*.py`
- `scripts/**/*.mjs`
- `requirements-*.txt`

## Git 中不建议保留

- 原始资料：`raw/`
- 预处理产物：`working/`
- 入库文档产物：`selected/`
- chunk 产物：`chunks/`
- 向量库：`vectors/`
- 模型文件：`models/`
- 临时输出：`outputs/`、`tmp/`
- 本地密钥：`.env`、证书、私钥

## 数据和模型版本建议

### 原始资料

建议使用独立台账记录：

- 文件名
- 来源部门
- 上传人
- 版本号
- 生效日期
- 是否入库
- 入库批次

### Chunk 和向量库

建议按批次管理，例如：

```text
batch_20260521
```

每个批次至少保留：

- `chunks/chunks.jsonl`
- `vectors/vector_manifest.json`
- `vectors/hybrid_manifest.json`
- `rag/hybrid_build_summary.md`

### 模型

建议模型放在服务器固定目录：

```text
/data/kb/models/bge-small-zh-v1.5
```

模型本身不进 Git，只在文档中记录：

- 模型名称
- 模型版本或下载来源
- embedding 维度
- 最大 token 长度
- 部署路径
- 上线时间

## 分支建议

- `main`：稳定版本，可部署。
- `dev`：日常开发和验证。
- `feature/*`：较大的功能改造，例如 `feature/embedding-service`。

## 提交建议

提交信息建议使用清晰前缀：

- `feat:` 新功能
- `fix:` 修复问题
- `docs:` 文档调整
- `build:` Docker、依赖、部署配置
- `chore:` 仓库整理、脚本小调整

示例：

```text
feat: add independent embedding service
docs: update repository structure guide
build: add dockerignore for knowledge artifacts
```

## 部署注意

Docker 构建上下文已经通过 `.dockerignore` 排除了大体量资料和向量库。部署时需要单独同步以下目录到 ECS：

```text
/data/kb/chunks
/data/kb/vectors
/data/kb/rag
/data/kb/selected
/data/kb/models
```

`kb-api` 读取这些目录，`embedding-service` 读取 `/data/kb/models`。
