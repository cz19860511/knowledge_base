# 平台事件回放机制

## 1. 目标

把任意一天的平台事件、日报和版本对账结果整理成一份可读的回放报告，方便排障、复盘和给大模型做上下文。

## 2. 回放内容

回放报告建议包含：

- 当天事件时间线
- 按事件类型的统计
- 按知识库的分组
- 当日报告内容
- 版本对账摘要

## 3. 当前实现

当前已提供：

- `GET /operations/replay-report`
- `scripts/generate_operation_replay_report.py`
- 回放 Markdown 输出到 `operations/replay/YYYY-MM-DD.md`

## 4. 使用方式

- 传 `date` 可回放指定日期。
- 传 `knowledge_base_id` 可只看某个知识库。
- 默认会保存 Markdown 文件，便于后续检索和入库。

## 5. 后续增强方向

- 增加按小时分段的时间线。
- 增加关键操作高亮。
- 增加与资产版本链的跳转链接。
- 增加首页回放入口和历史查询页。
