# 版本对账机制

## 1. 目标

把“操作事件”和“资产版本链”对起来，确认每次操作有没有留下对应资产、有没有漏记、有没有孤儿资产。

## 2. 对账范围

当前先对这些链路做对账：

- 原始文件上传 / 回滚 / 删除
- 预处理产物
- selected 主版本和候选清单
- chunk 产物
- 向量产物
- 平台运行日报和平台运行记忆库产物

## 3. 对账逻辑

- 按同一天的操作事件和资产台账生成对账结果。
- 事件里的 `input_assets` 和 `output_assets` 优先拿来匹配资产台账。
- 如果事件引用的文件在资产台账里找不到，就记为“缺失引用”。
- 如果资产台账里的记录找不到对应事件，就记为“孤儿资产”。

## 4. 输出内容

对账报告至少包含：

- 事件总数
- 资产总数
- 已关联引用数
- 缺失引用数
- 孤儿资产数
- 缺失引用清单
- 孤儿资产清单

## 5. 当前实现

当前已提供：

- `GET /operations/version-reconciliation`
- 对账 Markdown 报告输出到 `operations/reconciliation/YYYY-MM-DD.md`

## 6. 后续增强方向

- 把每条资产写入时绑定 `operation_id`
- 让对账从“按文件路径匹配”升级成“按事件 ID 精确匹配”
- 增加对账失败告警和首页提示
