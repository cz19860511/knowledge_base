# 知识平台自进化建议模板集

## 1. 目标

把平台运行事件进一步整理成统一的建议模板，方便大模型、人工和程序在同一套语义下理解“该优化什么、该回滚什么、该补什么资料”。

## 2. 模板总览

当前模板分成 3 类：

- 参数优化建议模板
- 回滚建议模板
- 补资料建议模板

这些模板既可以给大模型直接生成建议时参考，也可以给运维和研发在人工确认时统一口径。

## 3. 参数优化建议模板

### 适用场景

- chunk 太碎或太长
- embedding 检索命中不稳定
- 混合检索排序不理想
- 某类问题召回率突然下降

### 建议输入

- 最近一段时间的失败事件
- 当前流程配置快照
- 相关知识库 ID
- 检索问题或命中样例

### 建议输出

- 问题描述
- 可调参数
- 建议调整方向
- 风险提示
- 验证方式

### 推荐提示词

> 请结合最新操作事件、流程配置和检索样例，判断哪些参数需要调整，并给出最小变更建议、风险和验证方式。

## 4. 回滚建议模板

### 适用场景

- 原始文件传错
- 预处理结果明显异常
- 新版本 chunk 或向量效果变差
- 回滚能快速恢复稳定状态

### 建议输入

- 出问题的文件或批次
- 当前版本与上一版本对比
- 回滚记录
- 关联操作事件

### 建议输出

- 回滚对象
- 回滚原因
- 推荐回滚到的版本
- 影响范围
- 人工确认项

### 推荐提示词

> 请基于版本历史和操作事件判断是否需要回滚，若需要请明确回滚对象、推荐版本、影响范围和确认项。

## 5. 补资料建议模板

### 适用场景

- 问题命中不足
- 某些场景资料缺失
- 规则不足以支撑回答
- 某个业务模块长期召回偏弱

### 建议输入

- 未命中的问题列表
- 当前知识库覆盖范围
- 高频失败场景
- 相关业务模块

### 建议输出

- 缺口场景
- 需要补充的资料类型
- 建议责任人/来源
- 优先级
- 补齐后验证方式

### 推荐提示词

> 请根据未命中问题和运行事件，识别知识缺口并给出需要补充的资料类型、优先级和验证方式。

## 6. 平台级落地建议

- 建议模板和建议结果分开存储，模板负责“标准化”，结果负责“实例化”。
- 模板应保持短、结构清楚、可追溯。
- 后续可把模板暴露成接口，供 Agent 或平台页面直接调用。
- 建议生成时必须保留事件证据和人工确认入口。
