# 知识平台自进化建议规范

## 1. 目标

把平台运行事件转成结构化建议，让大模型和人工都能快速判断下一步应该优化什么。

## 2. 建议字段

建议项应包含：

- `suggestion_id`
- `category`
- `title`
- `summary`
- `recommendation`
- `evidence`
- `scope`
- `risk_level`
- `priority`
- `requires_human_confirmation`
- `related_event_types`
- `related_knowledge_base_ids`

## 3. 建议分类

建议分类建议统一为：

- `pipeline_stability`
- `data_governance`
- `knowledge_base_governance`
- `memory_sync`
- `platform_health`

## 4. 建议原则

- 先写事实，再给建议。
- 建议必须能追溯到具体事件证据。
- 建议默认都要人工确认。
- 建议要尽量短，避免一条建议描述太多动作。

## 5. 输出形式

建议可以输出成：

- JSON，方便程序消费
- Markdown，方便人和大模型阅读

## 6. 当前实现

当前已实现一个基于操作事件的规则版建议生成器，可作为后续大模型生成建议的底座。
